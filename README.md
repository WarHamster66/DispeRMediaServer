# DispeR Media Server

Телеграм-бот для управления домашним медиасервером на Linux: торренты, файловый
менеджер, мониторинг системы и ежедневные отчёты — всё из чата Telegram.

Под капотом: **Transmission** (торренты) · **Samba** (сетевая шара) · **Plex**
(медиатека, опционально) · **Python** + [pyTelegramBotAPI](https://github.com/eternnoir/pyTelegramBotAPI).

## Возможности

- 🎬 **Торренты** — добавление `.torrent`-файлов и magnet-ссылок прямо в чат,
  проверка свободного места перед загрузкой, прогресс в реальном времени,
  пауза/возобновление, история загрузок.
- 📂 **Файловый менеджер** — просмотр, удаление, переименование, перемещение
  файлов и папок через inline-кнопки (с защитой от выхода за пределы шары).
- 📊 **Мониторинг** — CPU, RAM, диски, температура CPU/HDD, аптайм, трафик,
  внутренний/внешний IP, состояние служб, скорость интернета.
- 🌦️ **Ежедневный отчёт** — настраиваемый по содержимому и времени, погода
  через Open-Meteo (бесплатно, без API-ключа — нужен только город).
- 🎞️ **Plex** — после докачки бот автоматически просит Plex пересканировать
  библиотеку, чтобы файл сразу появился (команда `/scan` — вручную).
- 🗄️ **Бэкап** — раз в неделю сохраняет архив `config.json` + `data/` (история,
  настройки) в папку на сервере (`BACKUP_DIR`, по умолчанию `~/media-server-backups`,
  хранит 8 последних). Вручную — команда `/backup`.
- 🔐 **Доступ** — команды доступны только пользователям из белого списка.

## Требования

- Ubuntu / Debian (тестировалось на 22.04 / 24.04)
- Python 3.10+
- Права root для установки (systemd, apt, transmission, samba)

## Установка

```bash
git clone https://github.com/WarHamster66/DispeRMediaServer.git
cd DispeRMediaServer
sudo python3 install.py
```

Установщик в интерактивном режиме:

1. Поставит системные пакеты (transmission-daemon, samba, smartmontools и др.).
2. Опционально установит Plex Media Server.
3. **Определит все диски, покажет свободное место и предложит, куда складывать
   загрузки** (можно выбрать по номеру или вписать свой путь).
4. Настроит Transmission RPC (включая правило AppArmor), Samba-шару с
   пользователем и паролем, общие права на медиапапку и Python-окружение (venv).
5. Спросит токен бота, Chat ID, ID администраторов, город для погоды и прокси.
6. Создаст и запустит systemd-сервис `media-server` с автозапуском.
7. Предложит закрепить **статический IP** (NetworkManager или netplan).

В конце выведет адрес сетевой папки с логином и паролем.

### Что подготовить заранее

| Что | Где взять |
|-----|-----------|
| Bot Token | [@BotFather](https://t.me/BotFather) |
| Chat ID / User ID | [@userinfobot](https://t.me/userinfobot) |

Погода работает без ключа (Open-Meteo) — нужно только указать город (`WEATHER_CITY` в `config.json`, установщик спросит его сам).

## Конфигурация

- **Секреты** (токены, пароли) — в `.env` (создаётся установщиком, права `600`,
  в git **не** попадает). Шаблон — `.env.example`.
- **Пути и настройки** — в `config.json` (медиапапка, разрешённые/защищённые
  папки, отслеживаемые службы, порог оповещения о месте на диске).

Изменить настройки после установки:

```bash
nano .env            # токены/пароли
nano config.json     # пути и поведение
sudo systemctl restart media-server
```

## Основные команды бота

| Команда | Действие |
|---------|----------|
| `/torrent` | добавить торрент (файл или magnet) |
| `/torrents` | активные загрузки |
| `/pause` · `/resume` | пауза / возобновление |
| `/report` | полный отчёт о сервере |
| `/configure` | настроить содержимое и время отчёта |
| `/dir` · `/dir2` · `/dir3` | файловый менеджер |
| `/media` | статистика медиатеки |
| `/history` | история загрузок |
| `/scan` | пересканировать библиотеку Plex |
| `/backup` | сохранить бэкап настроек на сервер |
| `/help` | все команды |

Также можно просто прислать боту `.torrent`-файл или magnet-ссылку.

```bash
systemctl status  media-server

journalctl -u media-server -f
```

## Решение проблем

### Торренты добавляются, но стоят на паузе (0%)

В Ubuntu демон Transmission ограничен профилем **AppArmor**, который запрещает
запись вне стандартных папок (`/var/lib/transmission-daemon/...`). При загрузке в
`/media`, `/srv` и т.п. в `dmesg` появляется `apparmor="DENIED" ... profile="transmission-daemon"`.
Установщик настраивает это автоматически, но если правка не применилась:

```bash
sudo tee /etc/apparmor.d/local/usr.bin.transmission-daemon > /dev/null <<'EOF'
/media/media-server/ rw,
/media/media-server/** rwk,
EOF
sudo apparmor_parser -r /etc/apparmor.d/usr.bin.transmission-daemon
sudo systemctl restart transmission-daemon
```

(замените `/media/media-server` на свой путь к медиапапке).

## Безопасность

- Не коммитьте `.env` — он в `.gitignore`.
- Команды бота ограничены белым списком `CREATOR_IDS`.
- Файловые операции ограничены папкой `SHARED_FOLDER` (защита от path traversal).
